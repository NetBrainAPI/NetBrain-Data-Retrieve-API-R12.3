# Introduction <a name="introduction"></a>
The API is used to get the default Geographic Hierarchy used by the Geographic traffic routing method from Azure API Server (https://management.azure.com/). The response includes the details like id, type, name, properties, etc.



The original Azure RESTful API used is <Geographic Hierarchies - Get Default>. 



For a complete list of available API resources for Azure Traffic Manager Profiles, please refer to Microsoft document: [Geographic Hierarchies - REST API (Azure Traffic Manager) | Microsoft Learn](https://learn.microsoft.com/en-us/rest/api/trafficmanager/geographic-hierarchies?view=rest-trafficmanager-2022-04-01)

For the original Azure API detailed information, please refer to Microsoft document: [Geographic Hierarchies - Get Default - REST API (Azure Traffic Manager) |](https://learn.microsoft.com/en-us/rest/api/trafficmanager/geographic-hierarchies/get-default?view=rest-trafficmanager-2022-04-01&tabs=HTTP#code-try-0)
