# Overview <a name="overview"></a>
The <b>API Gateway 5XX Error Rate</b> metric tracks the number of server-side errors (HTTP 5XX responses) generated by Amazon API Gateway. These errors typically indicate backend failures, integration issues, misconfigured endpoints, or internal service problems. Monitoring this metric is crucial for ensuring API reliability and quickly diagnosing issues caused by backend systems rather than client-side behavior.

# Metric Info <a name="metric-info"></a>
* <b>Metric Name</b>: 5XXError
* <b>Namespace</b>: AWS/ApiGateway
* <b>Unit / Stat</b>: SampleCount
* <b>Display Name in NetBrain</b>: 5XX Error Rate

# User-Defined Parameters <a name="user-defined-parameters"></a>
* <b>Start Time / End Time</b>: Define the time range to analyze data points, useful for historical analysis and real-time monitoring.
* <b>Default</b>: Last 24 hours
* <b>Statistics</b>: Default value is Count.
  * <b>Average</b>: Helps identify trends in backend error frequency over time.
  * <b>Sum</b>: Shows the total number of server-side errors within the specified period.
  * <b>Minimum</b>: Indicates periods with zero backend failures.
  * <b>Maximum</b>: Highlights significant backend failures or outage spikes.
* <b>Period</b>: Default value is 3600 second.
  * <b>Recommended Values</b>:
    * <b>60 seconds</b> for real-time detection of backend failures.
    * <b>300 seconds</b> for standard operational visibility.
    * <b>3600 seconds</b> for high-level system health analysis across extended periods.

# Use Cases Example <a name="example"></a>
## Use Case 1: Detecting Backend System Failures <a name="example-1"></a>
Track the <b>5XXError</b> metric to identify failures caused by integration endpoints, Lambda timeouts, internal server errors, or misconfigured backend connections. A spike in this metric often points to issues in downstream services.


## Use Case 2: Monitoring API Stability During Deployments <a name="example-2"></a>
During new API releases or backend updates, an increase in 5XX errors may indicate regression issues, incorrect configurations, or unstable backend components. Monitoring this metric helps ensure smooth and reliable deployments.
